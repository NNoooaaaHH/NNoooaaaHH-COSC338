Simplicity and Similarities Between 3D and 2D Renders

Before I even had the basic understanding of computer graphics pipelining I have always been under the assumption that the graphics pipeline was an extremely complicated process that would require many different steps and large amounds of code. I've always assumed the difference between rendering something simple such as a 2D polygon would be vasty different than rendering something complex like a 3D model. However, after learning more about the graphics pipeline and familiarizing myself with the process of graphics rendering, I have came to realize that the process of rendering a simple 2D polygon and a complex 3D model is actually suprisingly very similar. Once you understand the core foundations behind the graphics pipeline, you will start to realize that the process is actually quite simple and is consistant across all types of renders. Just look at the step by step process of rendering something simple such as a 2D triangle and something more complex like a 3D Model.

The pipeline structure is similar across many types of renders. They all go through the same steps: Vertex Creation > Vertex Shader > Rasterization > Fragment Shader > Display on Canvas The main differences being complex 3D renders involve more vertices and triangles when rending more detailed geometry. They also require more complex mathematics in both the vertex processing and the fragment shaders. However, the GPU stages remain similar and consistant. The code is also very consistant and similar between complex 3D renders and simple 2D renders. I was able to use the same code structure from rendering a simple red 2D triangle as a foundation to render something as complex as a 3D bunny model.

Mostly Math and Vertices

While working around with the code for rendering 2D polygons to make the 3D bunny model, I came to realize that much of what I was doing was just manipulating vertices and using math to get the vertices to be in the right place. It was both intresting and suprising to see how heavily graphics rendering relies on math equations. A large portion of the work I did invovled plugging values into equations, applying transformations, and adjusting vertex positions. I found it interesting that much of the complexity of rendering a 3D model comapred to a simple 2D shape comes mainly from the mathematics and the vertex manipulation, and not the graphics pipelining. In many ways rendering the 3D model felt like I was rendering a 2D shape but with an extra axis to work from.
